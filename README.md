# cnsite-bwin-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态形式存放，不针对任何特定域名或具体网站，仅供浏览与参考。

## 项目简介

该仓库主要维护一批独立 HTML 文件，每个文件包含独立的内容或功能。仓库本身不提供动态服务，所有页面均为纯前端实现，可直接在浏览器中打开使用。

适合需要集中存放静态页面、进行版本管理或共享 HTML 资源的场景。

## 目录说明

```
.
├── README.md          # 本文件
├── index.html         # 入口页面（可选）
├── pages/             # 页面归档目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/            # 公共资源（CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- `pages/`：存放各独立 HTML 页面的目录。
- `assets/`：存放页面引用的公共样式、脚本或图片资源。

## 页面归档说明

所有页面均为独立 HTML 文件，不依赖后端服务。每个页面均可在本地直接打开，或通过静态托管服务访问。

页面内容可能涉及不同主题或功能，具体请查看每个文件内的说明。建议在提交新页面时，保持文件命名清晰、结构一致。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并在必要时更新 `index.html` 中的导航链接。
- 修改页面：直接编辑对应 HTML 文件，注意保持与其他页面的兼容性。
- 删除页面：移除对应文件及引用链接。
- 资源管理：公共资源请统一放在 `assets/` 目录下，避免重复引用。

欢迎提交 Issue 或 Pull Request 进行讨论与改进。
